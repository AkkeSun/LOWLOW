[[common]]
== 공통 사항

=== HTTP Method
전송 요청을 위한 HTTP Method 방식을 제공하며, 전송 응답은 JSON 방식을 사용합니다.
|===
| Method | Tag name | Description

| `GET`
| `XXX-page`
| `데이터를 페이지 형태로 가져옵니다`

| `GET`
| `XXX-get`
| `한 개 데이터를 가져옵니다`

| `POST`
| `XXX-create`
| `새로운 데이터를 등록합니다`

| `UPDATE`
| `XXX-update`
| `기존 데이터를 수정합니다`

| `DELETE`
| `XXX-delete`
| `기존 데이터를 삭제합니다`
|===

=== Role
API 를 사용할 때 인증을 필요로 하며 권한에 따라 사용가능한 API가 제한됩니다.
|===
| ROLE | 사용 가능한 API | note

| `N/A`
| `calendar, gallery, notice, weekly, worshipVideo, basicInfo`
| `해당 API를 읽기만 가능합니다`

| `ROLE_UPLOADER`
| `calendar, gallery, notice, weekly, worshipVideo`
| `N/A 의 권한을 포함합니다`

| `ROLE_ADMIN`
| `accounting, basicInfo, member, memberAttend`
| `N/A, ROLE_UPLOADER 의 권한을 포함합니다`

| `ROLE_DEV`
| `jwt`
| `모든 권한을 갖습니다`
|===


=== Header
공통된 Header 값을 주고 받습니다.
|===
| Type | Header | Description | note

| `Request`
| `Accept`
| `application/hal+json;charset=UTF-8`
|

|
| `Content-Type`
| `application/json;charset=UTF-8`
| `Body 값을 서버에 전송할 때 사용됩니다`

|
| `ACCESS-TOKEN`
| `JWT Authentication Token`
| `인증이 필요한 API에 접근할 때 사용됩니다`

|
| `REFRESH-TOKEN`
| `JWT Authentication Token`
| `ACCESS-TOKEN 만료시 재발급에 사용됩니다`

| `Response`
| `Content-Type`
| `application/hal+json;charset=UTF-8`
|
|===
